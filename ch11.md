## はじめに

録音した音声にノイズが混ざっていることはよくあります。マイクのホワイトノイズ、空調の音、環境音などです。

この章では最も基本的なノイズ除去手法「スペクトル減算」を実装します。そしてその限界を実験で確認します。

---

## 必要なライブラリ

```python
import numpy as np
import matplotlib.pyplot as plt
from scipy.signal import stft, istft
from scipy.io.wavfile import write, read
```

---

## スペクトル減算の仕組み

スペクトル減算の考え方はシンプルです。

1. 音声をSTFTで周波数領域に変換する
2. ノイズだけが存在する区間（無音部分）のスペクトルを推定する
3. 全体のスペクトルからノイズスペクトルを引く
4. 逆変換（ISTFT）して時間領域に戻す

---

## テスト音声の準備

```python
sr = 16000

# クリーン音声を生成
t = np.linspace(0, 2.0, int(sr * 2.0), endpoint=False)
clean = (np.sin(2 * np.pi * 440 * t) +
         0.5 * np.sin(2 * np.pi * 880 * t))
clean = (clean / np.max(np.abs(clean))).astype(np.float32)

# ホワイトノイズを付加
np.random.seed(42)
noise_scale = 0.1
noisy = clean + np.random.normal(0, noise_scale, len(clean)).astype(np.float32)

# wavとして保存
write("clean.wav", sr, (clean * 32767).astype(np.int16))
write("noisy.wav", sr, (noisy * 32767).astype(np.int16))

# SNRを計算
signal_power = np.mean(clean ** 2)
noise_power = np.mean((noisy - clean) ** 2)
snr = 10 * np.log10(signal_power / noise_power)
print(f"SNR: {snr:.1f}dB")
```

## スペクトル減算の実装

```python
def spectral_subtraction(y, sr, noise_duration=0.3, hop=256, n_fft=1024):
    """
    スペクトル減算によるノイズ除去

    Parameters
    ----------
    y : 入力音声
    sr : サンプリング周波数
    noise_duration : ノイズ推定に使う先頭区間の長さ（秒）
    """
    # STFT
    _, _, Zxx = stft(y, fs=sr, nperseg=n_fft, noverlap=n_fft-hop)
    mag = np.abs(Zxx)
    phase = np.angle(Zxx)

    # ノイズ推定（先頭N秒をノイズのみの区間と仮定）
    noise_frames = max(1, int(noise_duration * sr / hop))
    noise_spec = np.mean(mag[:, :noise_frames], axis=1, keepdims=True)

    # スペクトル減算（負の値はゼロにクリップ）
    mag_denoised = np.maximum(mag - noise_spec, 0)

    # ISTFT で時間領域に戻す
    _, y_out = istft(mag_denoised * np.exp(1j * phase),
                     fs=sr, nperseg=n_fft, noverlap=n_fft-hop)
    return y_out[:len(y)].astype(np.float32)
```

## スペクトル減算の限界

実験でよく見られる問題が2つあります。

**問題1：ミュージカルノイズ**

ノイズ除去後に「ビリビリ」「チリチリ」という不自然な音が残ることがあります。

```python
# ノイズ強度を変えて効果を確認
print("=== ノイズ強度と除去後の品質 ===")
for scale in [0.02, 0.05, 0.10, 0.20]:
    np.random.seed(42)
    y_noisy = clean + np.random.normal(0, scale, len(clean)).astype(np.float32)
    y_denoised = spectral_subtraction(y_noisy, sr)

    # 元音声との残差（小さいほど品質が高い）
    residual = np.sqrt(np.mean((clean - y_denoised[:len(clean)]) ** 2))
    snr_in = 10 * np.log10(np.mean(clean**2) / (scale**2))
    print(f"noise_scale={scale:.2f} (SNR={snr_in:.0f}dB): "
          f"残差={residual:.4f}")
```

**問題2：ノイズ区間がないと逆効果**

スペクトル減算は「先頭がノイズのみの区間」という前提で動きます。

```python
# 音声の先頭からすぐ信号がある場合のシミュレーション
y_no_silence = noisy.copy()  # 先頭から音声信号がある

# 先頭0.3秒のスペクトルを確認
n_fft = 1024
hop = 256
_, _, Zxx = stft(y_no_silence, fs=sr, nperseg=n_fft, noverlap=n_fft-hop)
mag = np.abs(Zxx)

noise_frames = int(0.3 * sr / hop)
noise_est = np.mean(mag[:, :noise_frames], axis=1)
total_signal = np.mean(mag, axis=1)
ratio = noise_est.mean() / total_signal.mean()

print(f"\nノイズ推定値 / 実際の信号の比率: {ratio:.3f}")
print("→ 1.0に近いほど、音声信号をノイズと誤認している")
print("→ この状態でスペクトル減算を行うと、音声信号自体が削られる")
```

## 改善策：過減算係数の調整

```python
def spectral_subtraction_v2(y, sr, noise_duration=0.3,
                              alpha=1.0, hop=256, n_fft=1024):
    """
    改良版スペクトル減算

    Parameters
    ----------
    alpha : 過減算係数（大きいほど強くノイズを除去するが音質が下がる）
    """
    _, _, Zxx = stft(y, fs=sr, nperseg=n_fft, noverlap=n_fft-hop)
    mag = np.abs(Zxx)
    phase = np.angle(Zxx)

    noise_frames = max(1, int(noise_duration * sr / hop))
    noise_spec = np.mean(mag[:, :noise_frames], axis=1, keepdims=True)

    # alpha で除去の強さを調整
    mag_denoised = np.maximum(mag - alpha * noise_spec, 0)

    _, y_out = istft(mag_denoised * np.exp(1j * phase),
                     fs=sr, nperseg=n_fft, noverlap=n_fft-hop)
    return y_out[:len(y)].astype(np.float32)

# alphaを変えて比較
print("=== alpha（過減算係数）の影響 ===")
for alpha in [0.5, 1.0, 1.5, 2.0]:
    y_d = spectral_subtraction_v2(noisy, sr, alpha=alpha)
    residual = np.sqrt(np.mean((clean - y_d[:len(clean)]) ** 2))
    print(f"alpha={alpha:.1f}: 残差={residual:.4f}")
```

## スペクトル減算が適している場面

スペクトル減算は以下の条件が揃っているときに有効です。

- 録音前に「無音区間」を数秒設けられる
- ノイズが定常的（常に同じ音）
- 多少の音質低下が許容できる

逆に、会話や突発的な音（拍手、笑い声）の混じる環境には向きません。
