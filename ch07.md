## はじめに

前章ではスペクトル減算を実装しました。しかし実際に試すと以下の問題が起きます。

- 音声まで削れてしまう
- 不自然な歪み（ミュージカルノイズ）が発生する
- ノイズ区間がないと逆効果になる

この章では、より実用的な手法である**Wienerフィルタ**を実装し、スペクトル減算と比較します。

---

## Wienerフィルタとは

Wienerフィルタは**SNR（信号対雑音比）に基づいて補正を行う手法**です。

スペクトル減算が「引く」のに対し、Wienerフィルタは「比率で制御」します。

- ノイズが強い周波数成分 → 強く抑える
- 音声が強い周波数成分 → できるだけ残す

フィルタの式はシンプルです。

$$H = \frac{P_{signal}}{P_{signal} + P_{noise}}$$

$P_{signal}$ が大きい（音声が支配的）→ $H$ が1に近づく → 音声を保持  
$P_{noise}$ が大きい（ノイズが支配的）→ $H$ が0に近づく → 強く抑制

## テスト音声の準備

```python
sr = 16000
t = np.linspace(0, 2.0, int(sr * 2.0), endpoint=False)

# クリーン音声（440Hz + 倍音）
clean = (np.sin(2 * np.pi * 440 * t) +
         0.5 * np.sin(2 * np.pi * 880 * t))
clean = (clean / np.max(np.abs(clean))).astype(np.float32)

# ノイズを付加
np.random.seed(42)
noise_scale = 0.1
noisy = clean + np.random.normal(0, noise_scale, len(clean)).astype(np.float32)

snr = 10 * np.log10(np.mean(clean**2) / noise_scale**2)
print(f"SNR: {snr:.1f}dB")
```

## Wienerフィルタの実装

```python
def wiener_filter(y, sr, noise_duration=0.3, hop=256, n_fft=1024):
    """
    Wienerフィルタによるノイズ除去

    Parameters
    ----------
    y : 入力音声（先頭noise_duration秒が無音区間であること）
    noise_duration : ノイズ推定に使う先頭区間の長さ（秒）
    """
    # STFT
    _, _, Zxx = stft(y, fs=sr, nperseg=n_fft, noverlap=n_fft-hop)
    mag = np.abs(Zxx)
    phase = np.angle(Zxx)

    # ノイズ推定（先頭N秒）
    noise_frames = max(1, int(noise_duration * sr / hop))
    noise_spec = np.mean(mag[:, :noise_frames], axis=1, keepdims=True)

    # パワースペクトル
    signal_power = mag ** 2
    noise_power = noise_spec ** 2

    # Wienerフィルタ係数
    H = signal_power / (signal_power + noise_power + 1e-10)

    # フィルタ適用
    mag_filtered = mag * H

    # ISTFT で時間領域に戻す
    _, y_out = istft(mag_filtered * np.exp(1j * phase),
                     fs=sr, nperseg=n_fft, noverlap=n_fft-hop)
    return y_out[:len(y)].astype(np.float32)
```

スペクトル減算との実装の違いは1箇所だけです。

```python
# スペクトル減算
mag_denoised = np.maximum(mag - noise_spec, 0)      # 引いてゼロクリップ

# Wienerフィルタ
H = signal_power / (signal_power + noise_power)     # 比率で制御
mag_filtered = mag * H
```

## 結果の比較

```python
from scipy.signal import stft, istft

def spectral_subtraction(y, sr, noise_duration=0.3, hop=256, n_fft=1024):
    _, _, Zxx = stft(y, fs=sr, nperseg=n_fft, noverlap=n_fft-hop)
    mag, phase = np.abs(Zxx), np.angle(Zxx)
    nf = max(1, int(noise_duration * sr / hop))
    noise_spec = np.mean(mag[:, :nf], axis=1, keepdims=True)
    mag_d = np.maximum(mag - noise_spec, 0)
    _, y_out = istft(mag_d * np.exp(1j * phase),
                     fs=sr, nperseg=n_fft, noverlap=n_fft-hop)
    return y_out[:len(y)].astype(np.float32)

denoised_ss = spectral_subtraction(noisy, sr)
denoised_w  = wiener_filter(noisy, sr)

fig, axes = plt.subplots(4, 1, figsize=(12, 10), sharex=True)
t_axis = np.arange(len(clean)) / sr

axes[0].plot(t_axis, clean,       linewidth=0.5)
axes[0].set_title("Clean")
axes[1].plot(t_axis, noisy,       linewidth=0.5, color='orange')
axes[1].set_title(f"Noisy（SNR={snr:.1f}dB）")
axes[2].plot(t_axis, denoised_ss, linewidth=0.5, color='red')
axes[2].set_title("スペクトル減算後")
axes[3].plot(t_axis, denoised_w,  linewidth=0.5, color='green')
axes[3].set_title("Wienerフィルタ後")

for ax in axes:
    ax.set_ylabel("Amplitude")
axes[3].set_xlabel("Time [s]")
plt.tight_layout()
plt.show()
```

## ノイズ強度を変えて比較する

```python
print(f"{'noise_scale':>12} {'SNR(dB)':>8} {'SS残差':>8} {'Wiener残差':>12}")
print("-" * 44)

for scale in [0.02, 0.05, 0.10, 0.20]:
    np.random.seed(42)
    y_n = clean + np.random.normal(0, scale, len(clean)).astype(np.float32)
```

## スペクトル減算 vs Wienerフィルタ

| 項目 | スペクトル減算 | Wienerフィルタ |
|------|--------------|--------------|
| 仕組み | ノイズスペクトルを引く | SNRで比率制御 |
| ミュージカルノイズ | 出やすい | 出にくい |
| 音質劣化 | 大きい | 小さい |
| 計算コスト | 低い | 低い（ほぼ同じ） |
| ノイズ区間なしの場合 | 逆効果になりやすい | やや頑健 |

残差の数値だけ見るとWienerフィルタの方が大きく見えますが、これは「ノイズを除去しすぎず音声を保護している」ためです。実際に聴き比べると、Wienerフィルタの方が自然に聞こえます。

---

## まとめ

Wienerフィルタのポイントは3つです。

1. 「引く」のではなく「SNRに応じた比率で残す」設計
2. ミュージカルノイズが出にくく音質が自然
3. 実装はスペクトル減算とほぼ同じ複雑さ

次章ではノイズ除去の効果を分類精度で定量的に検証します。
