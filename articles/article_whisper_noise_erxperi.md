## はじめに

会議音声を自動で議事録にするツールをPythonで実装したとき、こんな疑問が浮かびました。

**「実際の会議室にはノイズがある。Whisper APIはどの程度のノイズに耐えられるのか？ノイズ除去処理を組み合わせれば精度は上がるのか？」**

この記事では、以下の内容を実験で検証した結果をまとめます。

- ノイズレベルを段階的に変えてWhisperの認識精度がどう変わるか
- スペクトルサブトラクション・Wienerフィルタ・RNNoiseの3手法でノイズ除去した場合の効果
- VAD（音声区間検出）が強ノイズ環境で機能するか

結論から言うと、**ソフトウェア的なノイズ除去には明確な限界があり、SNR -7dB程度の強ノイズ環境では試した手法がすべて逆効果**という結果になりました。

## 実験環境

```

```

使用した音声：自作の模擬会議音声（約30秒）
使用したノイズ：環境ノイズ（ambient noise）

## Step 1：ノイズレベルとWhisper精度の関係

### 実験方法

pydubで音声にノイズを付加し、4段階のノイズレベルでWhisper APIの文字起こし結果を比較しました。

```python
def mix_noise(speech_path, noise_path, noise_db, output_path):
    speech = AudioSegment.from_file(speech_path)
    noise  = AudioSegment.from_file(noise_path)
    while len(noise) < len(speech):
        noise = noise + noise
    noise = noise[:len(speech)]
    noise = noise + noise_db  # dBで音量調整
    mixed = speech.overlay(noise)
    mixed.export(output_path, format="wav")
```

### 結果

| ノイズレベル | 想定環境 | 認識結果 |
|------------|---------|---------|
| なし（ベースライン） | - | 正確（固有名詞のみ誤変換） |
| +2dB | 音声よりわずかにノイズ大 | 意味は通じるが誤変換多数 |
| +6dB | 工事現場程度 | +2dBとほぼ同じ結果 |
| +12dB | 限界テスト | 完全に崩壊・ハルシネーション発生 |

### 重要な発見：劣化は連続的ではなく閾値的に起きる

+2dBと+6dBで結果がほぼ同一だったことが興味深い点です。人間の聴覚は+2dBから+6dBにかけてじわじわ劣化しますが、Whisperは+2dBの時点ですでに誤認識モードに切り替わっていました。

実際に+2dBの音声を人間が聴いてみると、ギリギリ聞き取れるレベルでした。しかしWhisperの出力は大きく乱れています。これは「人間はコンテキストや予測を使って補完しながら聴いている」のに対し、**Whisperは音響的な特徴量に基づいて判断するため、音声信号が変質した帯域で誤認識が発生しやすい**ことを示しています。

## Step 2：周波数特性の分析

なぜこのノイズがWhisperに対して強く影響するのか、`scipy.signal.welch`を使って周波数特性を可視化しました。

```python
def plot_spectrum(data, rate, label):
    f, psd = signal.welch(data, fs=rate, nperseg=1024)
    psd_db = 10 * np.log10(psd + 1e-10)
    plt.plot(f, psd_db, label=label)
```

### 結果

```
0〜8kHz中、ノイズが優勢な帯域: 59.1%
音声主要帯域（300〜3400Hz）の平均SNR: -7.1dB
```

![スペクトル分析結果](/images/spectrum_analysis.png)

音声主要帯域での平均SNRが **-7.1dB** という結果が出ました。これはノイズが音声の約2倍以上の強さであることを意味します。この数字が後の実験結果を理解する上での鍵になります。

## Step 3：ノイズ除去手法の比較

### 試した手法

**① スペクトルサブトラクション（SS）**

古典的な手法。STFTで周波数領域に変換し、推定ノイズを差し引く。

```python
noise_mag   = np.mean(np.abs(Znoise), axis=1, keepdims=True)
cleaned_mag = np.maximum(signal_mag - noise_mag * 1.5, 0.0)
```

**② Wienerフィルタ**

各周波数でSNRを推定し、音声が残る確率を最大化する重みをかける。SSより理論的に優れた手法。

```python
snr_est = np.maximum(obs_power - noise_power, 0) / (noise_power + 1e-10)
H = np.maximum(snr_est / (snr_est + 1), floor)
```

ミュージカルノイズ対策として時間方向のスムージングも追加。

```python
from scipy.ndimage import uniform_filter1d
H = uniform_filter1d(H, size=5, axis=1)
```

**③ RNNoise（pyrnnoise）**

Mozillaが音声通話向けに開発した深層学習ベースのノイズ除去。torch不要で動作。

### 結果

| 手法 | Whisper認識精度 | 聴感 |
|------|--------------|------|
| ノイズあり +2dB | 意味は通じる | ギリギリ聞き取れる |
| SS後 | 悪化 | 聞き取りにくい |
| Wienerフィルタ後 | 悪化 | 聞き取りやすい |
| RNNoise後 | 悪化（SSより改善） | 聞き取りにくい |

### 考察：聴感とWhisperの乖離

**Wienerフィルタは聴感を改善したにもかかわらず、Whisperの認識精度は悪化しました。**

これは「人間の聴覚に自然に聞こえる音」と「Whisperが認識しやすい音」が必ずしも一致しないことを示しています。

Wienerフィルタは人間の聴覚に不快なSNRが低い帯域を大きく減衰させます。しかしWhisperはその帯域（フォルマント・子音の摩擦音）を使って認識するため、WienerフィルタがかえってWhisperの手がかりを削ってしまいました。

### なぜどの手法も逆効果なのか

SSもRNNoiseも+2dBと+6dBで同じ結果になりました。これは「ノイズ除去処理自体が別の歪みを生んでいる」ことを示しています。

SNR -7.1dBという条件では、ノイズ除去処理が音声成分を保持しながらノイズを除去するのに必要な情報（音声とノイズの周波数的な区別）が失われています。ノイズ除去処理は「ノイズを除去した」のではなく「音声を別の形で劣化させた」状態を作り出していたのです。

## Step 4：VADの動作検証

### 目的

ノイズ除去の前処理としてVAD（Voice Activity Detection）を使えないか検討しました。具体的には以下の2つの用途を想定しました。

- **ノイズプロファイルの精度向上**：無音区間（純粋なノイズのみの区間）を確実に検出してノイズ推定に使う
- **Whisperへの入力改善**：音声区間のみを渡すことで、ノイズ区間でのハルシネーションを防ぐ

### 実験方法

```python
import webrtcvad
vad = webrtcvad.Vad(aggressiveness=1)  # 0〜3
```

まずノイズあり音声に対してVADをかけ、そもそも強ノイズ環境で音声区間を正しく検出できるかを確認しました。

### 結果

| 条件 | 音声と判定された区間の割合 |
|------|----------------------|
| クリーン音声 | 57.1% |
| ノイズあり +2dB | 99.8% |
| Wiener後 | 96.2% |

クリーンな音声では全体の57.1%が音声区間として正しく検出されました。しかしSNR -7.1dBの環境では99.8%が「音声あり」と判定され、VADはほぼ機能しませんでした。

webrtcvadはエネルギーや周波数の形状に基づいて判定しますが、ノイズのエネルギーが音声と同程度以上ある環境では全フレームを音声と誤判定してしまいます。これはwebrtcvadが想定している使用環境（通話品質程度のノイズ）を大きく超えた条件です。

Wiener処理後でも96.2%と改善幅は小さく、ノイズ除去をVADの前処理として使っても実用的な効果は得られませんでした。強ノイズ環境ではVADの前提条件である「音声と無音の区別」自体が困難であることが確認されました。

## 総合考察

今回の実験で明らかになったことを整理します。

1. Whisperは音声主要帯域のSNRが負になると認識精度が閾値的に劣化する
2. その劣化は人間が聴いても同様で、音声信号自体の変質が原因
3. スペクトルサブトラクション・Wienerフィルタ・RNNoiseはいずれもSNR -7dBの強ノイズには逆効果
4. VADも強ノイズ環境では機能しない

**「聴感的に改善したのに認識精度が下がった」というWienerフィルタの結果は特に示唆的です。** 音声認識システムの前処理として使うノイズ除去は、人間の聴覚ではなくASRシステムの特性に合わせて設計する必要があります。

## おわりに

「Whisper APIに渡す前にノイズを除去すれば精度が上がる」という仮説を検証しましたが、強ノイズ環境ではむしろ逆効果という結果になりました。
