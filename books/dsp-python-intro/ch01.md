# Pythonで理解するサンプリング定理  
## はじめに

デジタル音声処理を学ぶと、最初に登場するのがサンプリング定理です。

「サンプリング周波数は信号の最大周波数の2倍以上必要」

教科書にはそう書いてあります。でも、なぜ2倍なのか。2倍を下回ると何が起きるのか。

この記事では、Pythonで実際に実験しながら確かめます。読み終えたあとには「2倍必要な理由」と「エイリアシングが起きる仕組み」が直感的に理解できるはずです。

## サンプリングとは何か

音声はアナログの連続信号です。これをデジタルで扱うには、一定間隔で値を取り出す必要があります。これがサンプリングです。

\`\`\`python
sr = 1000        # サンプリング周波数（1秒間に1000回測定）
t = np.linspace(0, 1, sr, endpoint=False)

f_signal = 5     # 信号の周波数（5Hz）
signal = np.sin(2 * np.pi * f_signal * t)

plt.figure(figsize=(10, 3))
plt.plot(t, signal)
plt.xlabel("Time [s]")
plt.ylabel("Amplitude")
plt.title("Original Signal (5Hz)")
plt.show()
\`\`\`
![](https://storage.googleapis.com/zenn-user-upload/3d19780f05a4-20260326.png)

\`sr=1000\` は「1秒間に1000回サンプリングする」という意味です。CDの音声は44100Hz、電話は8000Hzです。

5Hzの信号は1波形が1/5秒なので、1波形あたり200点データを持つことができます。
そのため、サンプリング周波数が1000Hzだと5Hzの波形は遜色なく描くことができます。

---

## ナイキスト周波数

サンプリング周波数を $f_s$ とすると、正しく表現できる最大周波数は

$$f_{Nyquist} = \frac{f_s}{2}$$

です。\`sr=1000Hz\` なら、500Hz以下の信号しか正確に表現できません。

なぜ2倍が必要なのか。直感的には「1周期を最低2点で捉えるには、その2倍の速さで測定する必要がある」からです。

---

## サンプリングを下げると何が起きるか

\`\`\`python
# 十分なサンプリング（ナイキスト条件を満たす）
fs_ok = 20       # 5Hzの信号に対して20Hz → 十分
t_ok = np.linspace(0, 1, fs_ok, endpoint=False)
sampled_ok = np.sin(2 * np.pi * f_signal * t_ok)

# 不十分なサンプリング（ナイキスト条件を満たさない）
fs_low = 8       # 5Hzの信号に対して8Hz → 不十分
t_low = np.linspace(0, 1, fs_low, endpoint=False)
sampled_low = np.sin(2 * np.pi * f_signal * t_low)

fig, axes = plt.subplots(1, 2, figsize=(12, 4))

axes[0].plot(t, signal, 'lightblue', label='Original')
axes[0].stem(t_ok, sampled_ok, 'C0', label='Sampled (20Hz)')
axes[0].set_title(f'OK: fs=20Hz (Nyquist={fs_ok//2}Hz > 5Hz)')
axes[0].legend()

axes[1].plot(t, signal, 'lightblue', label='Original')
axes[1].stem(t_low, sampled_low, 'C3', label='Sampled (8Hz)')
axes[1].set_title(f'NG: fs=8Hz (Nyquist={fs_low//2}Hz < 5Hz)')
axes[1].legend()

plt.tight_layout()
plt.show()
\`\`\`

サンプリング点が少ないと、元の波形を正しく再現できていないことが分かります。

## エイリアシングを実際に確認する

ナイキスト周波数を超える信号をサンプリングすると、「別の周波数として観測される」現象が起きます。これがエイリアシングです。

\`\`\`python
fs_sample = 20   # サンプリング周波数

# ナイキスト周波数を超える信号（15Hz）
f_alias = 15
t_low = np.linspace(0, 1, fs_sample, endpoint=False)
alias_signal = np.sin(2 * np.pi * f_alias * t_low)

# エイリアス周波数を計算
f_alias_result = abs(f_alias - round(f_alias / fs_sample) * fs_sample)
print(f"15Hzをfs={fs_sample}Hzでサンプリング")
print(f"→ エイリアス周波数: {f_alias_result}Hz")
# 出力: エイリアス周波数: 5Hz
\`\`\`

15Hzの信号をサンプリングすると、5Hzとして観測されます。元の信号が「化けて」しまうのです。

スペクトルで確認しましょう。

\`\`\`python
t_ref = np.linspace(0, 1, fs_sample, endpoint=False)
signal_5hz  = np.sin(2 * np.pi * 5  * t_ref)
signal_15hz = np.sin(2 * np.pi * 15 * t_ref)

fig, ax = plt.subplots(figsize=(10, 4))

# 連続波形を背景に薄く表示
t_cont = np.linspace(0, 1, 1000)
ax.plot(t_cont, np.sin(2 * np.pi * 5  * t_cont), 'C0', alpha=0.3)
ax.plot(t_cont, np.sin(2 * np.pi * 15 * t_cont), 'C3', alpha=0.3)

# サンプリング点を重ねてプロット
ax.scatter(t_ref, signal_5hz,  color='C0', s=100, zorder=3, label='5Hz')
ax.scatter(t_ref, signal_15hz, color='C3', s=40,  zorder=4, marker='x',
           linewidths=2, label='15Hz')

ax.axhline(y=0, color='gray', linewidth=0.5, linestyle='--', alpha=0.5)
ax.set_xlabel("Time [s]")
ax.set_ylabel("Amplitude")
ax.legend()
plt.tight_layout()
plt.show()
\`\`\`
![](https://storage.googleapis.com/zenn-user-upload/bec34eb7a4f8-20260326.png)

5Hzと15Hzのサンプリング点が完全に一致しています。デジタルでは区別できないのです。

## 実際の音声で体験する

サンプリング周波数を下げると、音がどう変わるか試してみましょう。

\`\`\`python
import numpy as np
from scipy.io.wavfile import write
from scipy.signal import resample_poly
from math import gcd

# テスト用音声を生成（440Hz + 6000Hz）
sr_orig = 44100
duration = 1.0
t = np.linspace(0, duration, int(sr_orig * duration), endpoint=False)
signal = (np.sin(2 * np.pi * 440  * t) +
          0.5 * np.sin(2 * np.pi * 6000 * t))
signal_int16 = (signal / np.max(np.abs(signal)) * 32767).astype(np.int16)
write("original.wav", sr_orig, signal_int16)

# 正しいダウンサンプリング（アンチエイリアシングフィルタ付き）
sr_low = 8000
g    = gcd(sr_orig, sr_low)
up   = sr_low  // g
down = sr_orig // g

signal_resampled = resample_poly(signal, up, down)
signal_low = signal_resampled.astype(np.float32)
write("downsampled.wav", sr_low,
      (signal_low / np.max(np.abs(signal_low)) * 32767).astype(np.int16))
\`\`\`

\`original.wav\` と \`downsampled.wav\` を聴き比べると、6000Hzの高い音成分だけが消えて、440Hzの音は保たれているのが分かります。

---

**注意：単純な間引きではダメな理由**

\`\`\`python
# ❌ 間違った実装（やってはいけない）
signal_wrong = signal_int16[::5]
write("wrong.wav", sr_orig // 5, signal_wrong)
# → アンチエイリアシングフィルタなし
# → 高周波成分が低周波に化けてノイズになる
# → しかも実際のサンプリング周波数は44100/5=8820Hzになる（8000Hzではない）
\`\`\`

正しいダウンサンプリングは「ローパスフィルタで高周波を除去してからサンプル数を変換する」の2ステップが必要です。\`resample_poly\` はこれを自動でやってくれます。

## まとめ

| 条件 | 結果 |
|------|------|
| $f_s \geq 2 f_{max}$ | 元の信号を正確に再現できる |
| $f_s < 2 f_{max}$ | エイリアシングが起きる |

サンプリング定理のポイントは3つです。

1. サンプリング周波数の半分（ナイキスト周波数）が表現できる最大周波数
2. それを超える信号は「別の周波数として化けて」観測される
3. CDが44100Hzなのは、人間の可聴域（最大約20000Hz）の2倍以上を確保するため
