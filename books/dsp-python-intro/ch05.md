## はじめに

前章でスペクトログラムを作りました。

しかし音声認識や音声分類では、スペクトログラムをそのまま使うことは少ないです。なぜなら人間の耳は、周波数を「線形」ではなく「対数的」に感じるからです。

100Hzと200Hzの違いは大きく聞こえますが、5000Hzと5100Hzの違いはほとんど感じません。

この人間の特性を取り入れた表現が「メルスペクトログラム」です。

## メル尺度とは何か

周波数をメル尺度に変換する式は次のとおりです。

$$m = 2595 \log_{10}\left(1 + \frac{f}{700}\right)$$

```python
def hz_to_mel(hz):
    return 2595 * np.log10(1 + hz / 700)

def mel_to_hz(mel):
    return 700 * (10 ** (mel / 2595) - 1)

# 周波数とメル値の対応を確認
freqs = [100, 500, 1000, 2000, 4000, 8000]
print(f"{'Hz':>8} {'Mel':>8}")
print("-" * 18)
for f in freqs:
    print(f"{f:>8}  {hz_to_mel(f):>8.0f}")
```

```
      Hz      Mel
------------------
     100      150
     500      607
    1000      999
    2000     1556
    4000     2146
    8000     2840
```

100→500Hzの差（400Hz）と、4000→8000Hzの差（4000Hz）が、メル尺度ではほぼ同じ間隔になっています。人間が「等間隔」に感じる尺度です。

## スペクトログラムとの違いを比較する

同じ音声から、線形スペクトログラムとメルスペクトログラムを並べて見てみましょう。

```python
# テスト音声の生成（複数の周波数成分を含む）
sr = 16000
duration = 2.0
t = np.linspace(0, duration, int(sr * duration), endpoint=False)

signal = (np.sin(2 * np.pi * 200  * t) +
          0.8 * np.sin(2 * np.pi * 800  * t) +
          0.5 * np.sin(2 * np.pi * 3000 * t) +
          0.3 * np.sin(2 * np.pi * 7000 * t))
signal = signal / np.max(np.abs(signal))

# STFT
n_fft = 1024
hop = 256
f_lin, t_s, Zxx = stft(signal, fs=sr, nperseg=n_fft, noverlap=n_fft-hop)
mag = np.abs(Zxx)

# 線形スペクトログラム
mag_db = 20 * np.log10(mag + 1e-10)

# メルフィルタバンクを作る
n_mels = 80
fmin, fmax = 0, sr // 2

mel_min = hz_to_mel(fmin)
mel_max = hz_to_mel(fmax)
mel_points = np.linspace(mel_min, mel_max, n_mels + 2)
hz_points = mel_to_hz(mel_points)
bins = np.floor((n_fft + 1) * hz_points / sr).astype(int)

filterbank = np.zeros((n_mels, n_fft // 2 + 1))
for m in range(1, n_mels + 1):
    for k in range(bins[m-1], bins[m]):
        filterbank[m-1, k] = (k - bins[m-1]) / (bins[m] - bins[m-1] + 1e-10)
    for k in range(bins[m], bins[m+1]):
        filterbank[m-1, k] = (bins[m+1] - k) / (bins[m+1] - bins[m] + 1e-10)

# メルスペクトログラム
mel_spec = np.dot(filterbank, mag ** 2)
mel_spec_db = 10 * np.log10(mel_spec + 1e-10)

# 並べて表示
fig, axes = plt.subplots(1, 2, figsize=(14, 5))

im1 = axes[0].pcolormesh(t_s, f_lin, mag_db, shading='gouraud',
                           vmin=-80, vmax=0, cmap='viridis')
axes[0].set_title("線形スペクトログラム")
axes[0].set_ylabel("Frequency [Hz]")
axes[0].set_xlabel("Time [s]")
plt.colorbar(im1, ax=axes[0], label="dB")

mel_axis = np.linspace(mel_min, mel_max, n_mels)  # メル値の軸
im2 = axes[1].pcolormesh(t_s, mel_axis, mel_spec_db, shading='gouraud',
                           vmin=-80, vmax=0, cmap='viridis')

# 縦軸の目盛りをメル値の位置に置き、ラベルはHz表示にする
tick_hz = [100, 500, 1000, 2000, 4000, 8000]
tick_mel = [hz_to_mel(f) for f in tick_hz if fmin <= f <= fmax]
tick_labels = [str(f) for f in tick_hz if fmin <= f <= fmax]
axes[1].set_yticks(tick_mel)
axes[1].set_yticklabels(tick_labels)

axes[1].set_title("メルスペクトログラム")
axes[1].set_ylabel("Frequency [Hz] (Mel scale)")
axes[1].set_xlabel("Time [s]")
plt.colorbar(im2, ax=axes[1], label="dB")

plt.tight_layout()
plt.show()
```

2つのグラフを見比べると、縦軸の目盛り間隔が違うことに気づきます。線形スペクトログラムでは目盛りが等間隔ですが、メルスペクトログラムでは低周波数域（100〜1000Hz）の間隔が広く、高周波数域（4000〜8000Hz）の間隔が狭くなっています。これがメルスケールの特徴で、人間が敏感な低周波数域の情報が拡大表示されます。

## librosaを使う場合

```python
import librosa
import librosa.display

signal, sr = librosa.load("sample.wav", sr=None)

mel_spec = librosa.feature.melspectrogram(
    y=signal, sr=sr,
    n_fft=1024,
    hop_length=256,
    n_mels=80
)
mel_spec_db = librosa.power_to_db(mel_spec, ref=np.max)

plt.figure(figsize=(10, 4))
librosa.display.specshow(mel_spec_db, sr=sr, hop_length=256,
                          x_axis='time', y_axis='mel')
plt.colorbar(format='%+2.0f dB')
plt.title("メルスペクトログラム (librosa)")
plt.tight_layout()
plt.show()
```

## n_melsを変えると何が変わるか

```python
for n_mels in [20, 40, 80, 128]:
    mel_spec = librosa.feature.melspectrogram(
        y=signal, sr=sr, n_mels=n_mels
    )
    print(f"n_mels={n_mels:3d}: 形状={mel_spec.shape}")
```

```
n_mels= 20: 形状=(20, 173)
n_mels= 40: 形状=(40, 173)
n_mels= 80: 形状=(80, 173)
n_mels=128: 形状=(128, 173)
```

n_melsは「メルフィルタの数」です。大きいほど周波数分解能が上がりますが、特徴量の次元も増えます。音声認識では80〜128、MFCCの計算には26程度が一般的です。

## なぜ音声AIでメルスペクトログラムが使われるのか

線形スペクトログラムとメルスペクトログラムを使って音声分類の精度を比べると、一般的にメルスペクトログラムの方が良い結果になります。

理由は2つあります。

一つ目は、人間の声の情報が低周波数域に集中しているからです。基本周波数（F0）は通常100〜400Hz、子音の特徴も1000Hz以下に多くあります。メルスケールはこの帯域を拡大して表現します。

二つ目は、次元削減の効果があるからです。線形スペクトログラムの高周波域（例：5000〜8000Hz）は人の声の分類にはほぼ不要な情報です。メルスペクトログラムはここを圧縮することで、重要な情報の比率を高めます。

## まとめ

| 項目 | 線形スペクトログラム | メルスペクトログラム |
|------|---------------------|---------------------|
| 周波数軸 | 線形（Hz） | 対数的（Mel） |
| 低周波数の表現 | 狭い | 広い |
| 音声AI向き | △ | ◎ |
| 計算コスト | 低い | 少し高い |
