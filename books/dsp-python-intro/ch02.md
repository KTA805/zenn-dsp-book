## はじめに

前の章でサンプリングを学びました。デジタル音声は「時間ごとの振幅値の列」として表現されています。

でも、音楽を聴くとき私たちは「時間」ではなく「音の高さ（周波数）」で音を認識しています。「ラの音（440Hz）」「ドの音（262Hz）」のように。

FFTは、時間領域の信号を周波数領域に変換する処理です。「この音にはどんな周波数がどれだけ含まれているか」を調べられます。

---

## 必要なライブラリ

```python
import numpy as np
import matplotlib.pyplot as plt
from scipy.fft import fft, fftfreq
from scipy.io.wavfile import write
```

---

## FFTの直感的な理解

FFTは「音を分解する処理」です。

虹がプリズムによって白色光を色に分解するように、FFTは音を周波数成分に分解します。

```python
sr = 1000
T = 1.0
t = np.linspace(0, T, sr, endpoint=False)

# 2つの音を混ぜる
f1, f2 = 50, 120
signal = np.sin(2 * np.pi * f1 * t) + 0.5 * np.sin(2 * np.pi * f2 * t)

plt.figure(figsize=(10, 3))
plt.plot(t[:200], signal[:200])
plt.xlabel("Time [s]")
plt.ylabel("Amplitude")
plt.title("混合信号（50Hz + 120Hz）― 時間領域では複雑に見える")
plt.show()
```

時間領域では複雑な波形に見えますが、FFTをかけると何が混ざっているかが分かります。

---

## FFTで周波数成分を取り出す

```python
N = len(signal)
Y = fft(signal)
freqs = fftfreq(N, 1/sr)

# 正の周波数のみ、振幅を正規化して表示
mask = freqs >= 0
magnitude = np.abs(Y[mask]) * 2 / N

plt.figure(figsize=(10, 4))
plt.plot(freqs[mask], magnitude)
plt.xlabel("Frequency [Hz]")
plt.ylabel("Magnitude")
plt.title("FFT結果 ― 50Hzと120Hzのピークが見える")
plt.xlim(0, 200)
plt.show()
```

50Hzと120Hzにピークが現れます。混ざっていた成分が分離されました。

ピークの高さが違うのは、元の信号で120Hzの振幅を0.5倍にしたからです。FFTは振幅の大きさも正確に反映します。

## 実際の音声ファイルを分析する

人工信号だけでなく、実際の音声で試してみましょう。

まず、分析用の音声を生成します。

```python
# 「ラ」の音（440Hz）に倍音を加えた音声を生成
sr_audio = 44100
duration = 1.0
t = np.linspace(0, duration, int(sr_audio * duration), endpoint=False)

# 基本音＋倍音（実際の楽器音に近い）
audio = (1.0 * np.sin(2 * np.pi * 440  * t) +   # 基本音
         0.5 * np.sin(2 * np.pi * 880  * t) +   # 第2倍音
         0.3 * np.sin(2 * np.pi * 1320 * t) +   # 第3倍音
         0.2 * np.sin(2 * np.pi * 1760 * t))    # 第4倍音

audio = (audio / np.max(np.abs(audio)) * 32767).astype(np.int16)
write("la_note.wav", sr_audio, audio)
```

このwavファイルをFFTで分析します。

```python
from scipy.io.wavfile import read

sr, data = read("la_note.wav")
data = data.astype(np.float32) / 32767  # 正規化

# FFT（計算速度のため先頭1秒だけ）
N = sr  # 1秒分
Y = fft(data[:N])
freqs = fftfreq(N, 1/sr)

mask = freqs >= 0
magnitude = np.abs(Y[mask]) * 2 / N

plt.figure(figsize=(12, 4))
plt.plot(freqs[mask], magnitude)
plt.xlabel("Frequency [Hz]")
plt.ylabel("Magnitude")
plt.title("「ラ」の音のFFT ― 440Hzと倍音が見える")
plt.xlim(0, 5000)
plt.ylim(0, 0.6)

# ピーク周波数に縦線を引く
for f, label in [(440, "440Hz"), (880, "880Hz"), (1320, "1320Hz"), (1760, "1760Hz")]:
    plt.axvline(x=f, color='red', linestyle='--', alpha=0.4)
    plt.text(f+30, 0.5, label, fontsize=9, color='red')

plt.show()
```

440Hzを基本音として、880・1320・1760Hzの倍音が並んで見えます。実際の楽器もこのような倍音構造を持っています。

## FFTの結果が持つ意味

```python
# FFTパラメータと周波数分解能の関係
N = 1024       # サンプル数
sr = 44100     # サンプリング周波数

freq_resolution = sr / N
print(f"サンプル数 N={N}")
print(f"周波数分解能: {freq_resolution:.1f}Hz")
print(f"→ {freq_resolution:.1f}Hz以上離れた周波数を区別できる")

# Nを変えると分解能が変わる
for n in [256, 512, 1024, 4096]:
    res = sr / n
    time_window = n / sr * 1000
    print(f"N={n:5d}: 周波数分解能={res:6.1f}Hz, 時間窓={time_window:.1f}ms")
```

```
N=  256: 周波数分解能= 172.3Hz, 時間窓=  5.8ms
N=  512: 周波数分解能=  86.1Hz, 時間窓= 11.6ms
N= 1024: 周波数分解能=  43.1Hz, 時間窓= 23.2ms
N= 4096: 周波数分解能=  10.8Hz, 時間窓= 92.9ms
```

Nを大きくすると周波数分解能が上がりますが、より長い時間窓が必要になります。時間分解能と周波数分解能はトレードオフの関係です。

## よくある疑問：なぜ複素数が出てくるのか

```python
Y = fft(signal)
print(type(Y[0]))      # complex128
print(Y[:3])           # 複素数の配列
```

FFTの結果は複素数です。これは振幅だけでなく「位相」も含んでいるためです。

音声分析では多くの場合、位相は無視して振幅だけを使います。

```python
magnitude = np.abs(Y)   # 振幅（複素数の絶対値）
phase = np.angle(Y)     # 位相（必要なときだけ使う）
```

## まとめ

FFTで分かることを整理します。

| 情報 | 取り出し方 |
|------|-----------|
| 含まれる周波数 | ピークの位置（x軸） |
| 各周波数の強さ | ピークの高さ（y軸） |
| 周波数分解能 | サンプリング周波数 / サンプル数 |

FFTは音声処理の出発点です。次の章では、時間とともに変化する周波数をとらえる「スペクトログラム」を作ります。
