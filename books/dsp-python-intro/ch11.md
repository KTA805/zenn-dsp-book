## はじめに

前回はスペクトル減算とWienerフィルタによるノイズ除去を実装しました。

ところで、こんな疑問を持ちませんでしたか？

> **「ノイズ除去すれば、音声分類の精度は上がるはずだよね？」**

実は、これは正しいとは限りません。

本記事では、実際に実験してその答えを確かめます。結果として出てきた数値は、最初の予想を裏切るものでした。

---

## 実験の設計

以下の3パターンで音声分類の精度を比較します。

- **Clean**：ノイズなしの音声でテスト
- **Noisy**：ノイズを加えた音声でテスト
- **Denoised**：ノイズ除去後の音声でテスト

学習は常にClean音声で行います。つまり「きれいな音声で学習したモデルが、現実のノイズ環境でどこまで通用するか」を測る実験です。

## 音声データの準備

実験用に、3種類の音（440Hz・880Hz・220Hz）を生成します。
母音「あ・い・う」を模したシンプルな信号です。

```python
import numpy as np
from scipy.signal import stft, istft
from scipy.fft import dct

np.random.seed(42)

def generate_tone(freq, sr=16000, duration=2.0):
    t = np.linspace(0, duration, int(sr * duration), endpoint=False)
    y = np.sin(2 * np.pi * freq * t)
    y += 0.5 * np.sin(2 * np.pi * freq * 2 * t)  # 倍音を加えて自然な音に
    y += 0.3 * np.sin(2 * np.pi * freq * 3 * t)
    return (y / np.max(np.abs(y))).astype(np.float32)

sr = 16000
sounds = {
    "a": generate_tone(440, sr),   # あ
    "i": generate_tone(880, sr),   # い
    "u": generate_tone(220, sr),   # う
}
```

## 音声分割

0.5秒ずつに分割してデータ数を増やします。

```python
def split_audio(y, sr, duration=0.5):
    samples = int(sr * duration)
    return [y[i:i+samples] for i in range(0, len(y)-samples+1, samples)]
```

## 特徴量抽出（MFCC）

librosaを使わず、scipy だけでMFCCを計算します。

```python
def mel_filterbank(sr, n_fft, n_mels=26):
    def hz_to_mel(hz): return 2595 * np.log10(1 + hz / 700)
    def mel_to_hz(mel): return 700 * (10 ** (mel / 2595) - 1)
    mel_points = np.linspace(hz_to_mel(0), hz_to_mel(sr/2), n_mels + 2)
    hz_points = mel_to_hz(mel_points)
    bins = np.floor((n_fft + 1) * hz_points / sr).astype(int)
    filters = np.zeros((n_mels, n_fft // 2 + 1))
    for m in range(1, n_mels + 1):
        for k in range(bins[m-1], bins[m]):
            filters[m-1, k] = (k - bins[m-1]) / (bins[m] - bins[m-1] + 1e-10)
        for k in range(bins[m], bins[m+1]):
            filters[m-1, k] = (bins[m+1] - k) / (bins[m+1] - bins[m] + 1e-10)
    return filters

def extract_mfcc(y, sr, n_mfcc=13, n_fft=512, hop=256):
    _, _, Zxx = stft(y, fs=sr, nperseg=n_fft, noverlap=n_fft-hop)
    power = np.abs(Zxx) ** 2
    fb = mel_filterbank(sr, n_fft)
    mel_power = np.dot(fb, power)
    mel_power = np.where(mel_power == 0, np.finfo(float).eps, mel_power)
    log_mel = np.log(mel_power)
    mfcc = dct(log_mel, type=2, axis=0, norm='ortho')[:n_mfcc]
    return mfcc

def extract_features(y, sr):
    mfcc = extract_mfcc(y, sr)
    return np.concatenate([np.mean(mfcc, axis=1), np.std(mfcc, axis=1)])
```

`mean`（平均）だけでなく`std`（標準偏差）も使うことで、
時間的な変化のパターンも特徴量に含められます。

## ノイズ付加とノイズ除去

```python
def add_noise(y, scale=0.05):
    return y + np.random.normal(0, scale, len(y)).astype(np.float32)

def spectral_subtraction(y, sr, hop=256, n_fft=512):
    _, _, Zxx = stft(y, fs=sr, nperseg=n_fft, noverlap=n_fft-hop)
    mag, phase = np.abs(Zxx), np.angle(Zxx)
    noise_frames = max(1, int(0.3 * sr / hop))
    noise_spec = np.mean(mag[:, :noise_frames], axis=1, keepdims=True)
    mag_d = np.maximum(mag - noise_spec, 0)
    _, y_out = istft(mag_d * np.exp(1j * phase), fs=sr,
                     nperseg=n_fft, noverlap=n_fft-hop)
    return y_out[:len(y)].astype(np.float32)

def wiener_filter(y, sr, hop=256, n_fft=512):
    _, _, Zxx = stft(y, fs=sr, nperseg=n_fft, noverlap=n_fft-hop)
    mag, phase = np.abs(Zxx), np.angle(Zxx)
    noise_frames = max(1, int(0.3 * sr / hop))
    noise_spec = np.mean(mag[:, :noise_frames], axis=1, keepdims=True)
    signal_power = mag ** 2
    noise_power = noise_spec ** 2
    H = signal_power / (signal_power + noise_power + 1e-10)
    _, y_out = istft(mag * H * np.exp(1j * phase), fs=sr,
                     nperseg=n_fft, noverlap=n_fft-hop)
    return y_out[:len(y)].astype(np.float32)
```

## 学習とテスト

```python
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import accuracy_score

label_map = {"a": 0, "i": 1, "u": 2}

# 学習データ（Clean音声）
X_train, y_train = [], []
for name, audio in sounds.items():
    for seg in split_audio(audio, sr):
        X_train.append(extract_features(seg, sr))
        y_train.append(label_map[name])

model = LogisticRegression(max_iter=1000, random_state=42)
model.fit(np.array(X_train), np.array(y_train))

# ノイズ強度を変えながら精度を比較
print(f"{'ノイズ強度':<12} {'Clean':>8} {'Noisy':>8} {'スペクトル減算':>14} {'Wiener':>8}")
print("-" * 56)

for scale in [0.01, 0.03, 0.05, 0.10, 0.20]:
    np.random.seed(99)
    xc, xn, xd_ss, xd_w, yt = [], [], [], [], []
    for name, audio in sounds.items():
        for seg in split_audio(audio, sr):
            yn = add_noise(seg, scale=scale)
            xc.append(extract_features(seg, sr))
            xn.append(extract_features(yn, sr))
            xd_ss.append(extract_features(spectral_subtraction(yn, sr), sr))
            xd_w.append(extract_features(wiener_filter(yn, sr), sr))
            yt.append(label_map[name])
    yt = np.array(yt)
    ac  = accuracy_score(yt, model.predict(np.array(xc)))
    an  = accuracy_score(yt, model.predict(np.array(xn)))
    ass = accuracy_score(yt, model.predict(np.array(xd_ss)))
    aw  = accuracy_score(yt, model.predict(np.array(xd_w)))
    print(f"scale={scale:<6}   {ac:>7.1%}  {an:>7.1%}  {ass:>13.1%}  {aw:>7.1%}")
```

## 予想と違った結果

正直に言うと、最初の予想はこうでした。

> Clean ≒ Denoised > Noisy

しかし実際は

> **Clean = Noisy > Wiener > スペクトル減算**

ノイズを加えても精度はほぼ落ちず、むしろノイズ除去した方が精度が下がりました。

---

## なぜこうなったのか

原因を調べてみると、構造的な問題が見えてきました。

```python
# スペクトル減算の内部を確認する
_, _, Zxx = stft(yn, fs=sr, nperseg=512, noverlap=256)
mag = np.abs(Zxx)
noise_frames = max(1, int(0.3 * sr / 256))

print(f"全フレーム数: {mag.shape[1]}")           # → 33
print(f"ノイズ推定フレーム数: {noise_frames}")   # → 18

noise_spec = np.mean(mag[:, :noise_frames], axis=1)
signal_spec = np.mean(mag, axis=1)
ratio = noise_spec / (signal_spec + 1e-10)
print(f"ノイズ推定値 / 信号の比率: {ratio.mean():.3f}")  # → 1.027
```

比率が1.027、つまりほぼ1.0です。

スペクトル減算は「先頭0.3秒はノイズだけの区間」と仮定してノイズを推定しますが、今回の音声には最初から音声信号が入っています。結果として、音声信号そのものをノイズとして推定し、それを差し引いてしまっていました。

ノイズ除去で精度が下がった理由は「ノイズが多かったから」ではなく、「ノイズ推定が根本的に間違っていたから」です。

---

## 3つの学び

**学び1：スペクトル減算は「ノイズ区間」が前提**

先頭に無音区間がない音声に適用すると、信号を削ってしまいます。
録音環境を設計できる場面（マイクON→数秒待つ→話す）でないと機能しません。

**学び2：Wienerフィルタは頑健**

同じノイズ推定の問題を抱えながらも、精度の落ち方が緩やかです。
これは「引く」のではなく「SNRに応じた比率で残す」設計のおかげです。
スペクトル減算より実用的な理由がここにあります。

**学び3：モデルの強さが前処理の効果を決める**

今回の実験では、ロジスティック回帰がMFCCでもノイズ音声を正確に分類できていました（Noisy=100%）。モデルが十分に強い場合、無理にノイズ除去をしても精度は上がりません。前処理はモデルの弱点を補うものです。

---

## まとめ

| 条件 | 精度（scale=0.05） |
|------|-------------------|
| Clean | 100.0% |
| Noisy | 100.0% |
| スペクトル減算後 | 50.0% |
| Wienerフィルタ後 | 100.0% |

ノイズ除去は「やれば良くなる」ものではありません。

- ノイズ区間の設計が正しくないと逆効果になる
- Wienerフィルタの方がスペクトル減算より実用的
- モデルが強ければ前処理なしでも精度は出る

音声処理では、**実験して数値で確認する**習慣が重要です。
直感と結果が一致しないことは、よくあります。

## 発展実験：実際のファンノイズで試す

ここまでの実験はホワイトノイズ（ランダムな雑音）を使っていました。しかし現実の録音環境では、PCのファンノイズや空調音のような「定常ノイズ」が混入することがほとんどです。

そこで、実際のPCファンを録音したwavファイルを使って同じ実験をしてみました。

---

### 設計の変更点

ホワイトノイズ実験との違いは2つです。

**① 先頭0.3秒を正しい無音区間にした**

```python
def generate_clean(freq, sr=16000, silence=0.3, duration=1.0):
    """先頭silence秒は無音、その後にトーンを配置する"""
    sil_len = int(sr * silence)
    tone_len = int(sr * duration)
    t = np.linspace(0, duration, tone_len, endpoint=False)
    tone = (np.sin(2 * np.pi * freq * t) +
            0.5 * np.sin(2 * np.pi * freq * 2 * t) +
            0.3 * np.sin(2 * np.pi * freq * 3 * t))
    tone = (tone / np.max(np.abs(tone))).astype(np.float32)
    y = np.zeros(sil_len + tone_len, dtype=np.float32)
    y[sil_len:] = tone
    return y
```

この設計により、スペクトル減算が「先頭0.3秒 = ノイズのみ」という前提を正しく満たせます。

**② ノイズをファンnoiseのwavから加算した**

```python
def add_fan_noise(y, fan_noise, gain=1.0):
    """fan_noiseをループして音声全体に加算する"""
    noise = np.tile(fan_noise, (len(y) // len(fan_noise)) + 1)[:len(y)]
    return (y + gain * noise).astype(np.float32)
```

`gain` を変えることでSNR（信号対雑音比）を調整します。

---

### 学び4：ノイズ区間の設計でスペクトル減算も機能する

ch10の前半実験では、スペクトル減算が精度を大きく下げました（50%）。原因は「ノイズ推定区間に音声信号が含まれていた」ことでした。

今回は先頭0.3秒を正しく無音区間にした結果、スペクトル減算はSNR=20dBまで100%を維持しました。

つまりスペクトル減算の問題は「アルゴリズムが弱い」のではなく「ノイズ推定の設計が間違っていた」ことにあります。

**録音時にマイクをONにしてから数秒待ってから話す**という習慣だけで、スペクトル減算の精度は大きく改善します。

ただし極端なノイズ（SNR≈0dB）ではスペクトル減算もWienerフィルタもノイズを除去しきれません。この場合はより高度な手法（深層学習ベースのノイズ除去など）が必要になります。

### まとめ：ノイズ除去の効果を決める3つの要因

1. **ノイズの種類**：定常ノイズ（ファン・空調）はスペクトル系手法が効く。突発的なノイズには弱い
2. **ノイズ区間の設計**：先頭に無音区間があるかどうかで精度が大きく変わる
3. **SNR（信号対雑音比）**：SNRが低すぎると（0dB以下）どの手法も限界がある
